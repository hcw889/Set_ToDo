# Set_ToDo (셋투두)

친구와 함께하는 **소셜 정산 투두** — 미니 데모.

매일 **출석**으로 하루를 시작하고, 미션을 수행한 뒤 **사진으로 인증**해 친구들의 승인을 받습니다.
못 지킨 미션은 벌금 포인트가 되어 모임통장(pool)에 쌓이고, 30일 뒤 모인 포인트를
**잘 지킨 사람**이 규칙대로 나눠 가집니다.

> DB·실인증·결제 없이 **보여주기(시연)** 목적. 현재 사용자의 진행 상황과 모임 구성은
> `localStorage`로 인터랙티브하게 동작하고, 친구들의 반응(출석·인증 승인·과거 기록)은
> 결정론적 목업으로 표현됩니다. (단일 기기 데모)

## 구성

| 폴더 | 역할 | 스택 |
|------|------|------|
| `client/` | 프론트엔드 (화면 + 상태) | React + Vite + TypeScript + Tailwind + React Router |
| `server/` | 목업 API + 확장 지점 | Express + TypeScript (DB 없음) |

### 데이터가 사는 곳

| 데이터 | 위치 | 비고 |
|--------|------|------|
| 현재 사용자, 진행 기록, 출석 | 브라우저 `localStorage` | 새로고침해도 유지 |
| 모임 · 인원 · 투두 | 브라우저 `localStorage` (`lib/groupStore.ts`) | **홈·모임·정산이 공유하는 단일 소스** |
| 기본 미션 · 챌린지 기간 | Express 목업 (`server/src/mock.ts`) | `client/src/lib/api.ts` 통해서만 호출 |
| 친구들의 과거 완료 기록 | `client/src/lib/mockProgress.ts` | userId 해시 시드 → 새로고침해도 동일 |
| 친구들의 출석 시각 | `client/src/features/attendance/mockCheckins.ts` | day + userId 해시 시드 |
| 친구들의 인증 카드 | `client/src/features/setlog/mockCards.ts` | 그라디언트 + 이모지 (외부 이미지 없음) |

## 실행

```bash
npm install        # 루트에서 (workspaces가 client/server 동시 설치)
npm run dev        # client(5173) + server(4000) 동시 기동
```

브라우저에서 http://localhost:5173 접속.

- `npm run dev:client` / `npm run dev:server` 로 개별 실행도 가능
- Vite dev 서버가 `/api/*` 요청을 Express(4000)로 프록시하므로 CORS 설정 불필요
- `host: true` 설정 → 같은 Wi-Fi의 휴대폰에서 `http://<PC-IP>:5173` 접속 가능 (사진 인증 시연용)
- 타입 확인: `cd client && npx tsc --noEmit`

## 데모 흐름

1. **이름 입력** (로그인 대체) → 대시보드 진입
2. **출석하기 👋** — 오늘의 미션 위에 출석 카드가 떠 있고, 누르기 전까지 미션은 잠겨 있음
   - 출석하면 미션이 열리고, 출석판에 내 아바타가 ✓ 로 표시됨
   - 연속 출석일(streak)이 함께 계산됨
3. **오늘의 출석판** — 모임 멤버 중 오늘 누가 시작했는지 / 몇 시에 했는지 한눈에
4. **미션 완료 → 사진 인증 (셋로그)**
   - `완료` 버튼 → 카메라 촬영 → 캡션 입력 → 인증 요청
   - 친구들이 2초 간격으로 하나씩 승인 → 전원 승인되면 미션 완료 처리
   - 인증 피드에서 **친구들의 인증 카드에 인증/반려**도 해볼 수 있음
5. 상단 **🛠️ 데모 도구**로 시간여행:
   - `다음 날로 ▶` — 하루 진행 (출석이 다시 잠기고, 미완료였던 과거 날은 벌금 확정)
   - `출석 다시하기 ↺` — 오늘 출석을 취소해 출석 버튼을 다시 띄움
   - `샘플 기록 채우기` — 12일치 기록 + 출석을 채우고 13일차로 점프
   - `30일 종료 시뮬레이션` — 최종 정산 보기
   - `진행 초기화` / `로그아웃`
6. **모임** 탭 — 모임 추가/삭제, 인원 추가/삭제 (최소 3명 · 최대 5명), 멤버별 30일 진행 그리드와 누적 벌금, 모임통장 합계
7. **정산** 탭 — `한 명 몰아주기` / `잘 지킨 만큼 분배` 규칙별 분배 결과

> **인원 동기화**: 모임 탭에서 인원을 추가/삭제하면 홈 출석판·인증 피드·정산에 **즉시** 반영됩니다.
> `useGroups()`는 `DemoContext` 한 곳에서만 호출하고, 나머지 화면은 컨텍스트를 통해 같은 값을 씁니다.
