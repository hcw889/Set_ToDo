# Set_ToDo (셋투두)

친구와 함께하는 **소셜 정산 투두** — 하루 완성 미니 데모.

친구가 정해준 미션을 매일 완료하고, 못 지키면 벌금 포인트가 모임통장(pool)에 쌓입니다.
30일 뒤 모인 포인트를 **잘 지킨 사람**이 규칙대로 나눠 가집니다.

> DB·실인증·결제 없이 **보여주기(시연)** 목적. 현재 사용자는 `localStorage`로 인터랙티브하게
> 동작하고, 친구들은 서버(Express) 목업 데이터로 표현됩니다. (단일 기기 데모)

## 구성

| 폴더 | 역할 | 스택 |
|------|------|------|
| `client/` | 프론트엔드 (화면 + 상태) | React + Vite + TypeScript + Tailwind + React Router |
| `server/` | 목업 API + 확장 지점 | Express + TypeScript (DB 없음) |

- 현재 사용자 진행상황 → 브라우저 `localStorage` (새로고침해도 유지)
- 친구/모임 데이터 → `server/src/mock.ts` (결정론적 시드)
- 프론트는 `client/src/lib/api.ts` 한 곳에서만 서버를 호출 → 나중에 실 백엔드로 교체 쉬움

## 실행

```bash
npm install        # 루트에서 (workspaces가 client/server 동시 설치)
npm run dev        # client(5173) + server(4000) 동시 기동
```

브라우저에서 http://localhost:5173 접속.

- `npm run dev:client` / `npm run dev:server` 로 개별 실행도 가능
- Vite dev 서버가 `/api/*` 요청을 Express(4000)로 프록시하므로 CORS 설정 불필요

## 데모 흐름

1. 이름 입력(로그인 대체) → 대시보드 진입
2. **오늘의 미션 완료 버튼** 클릭 → 완료율/벌금 반영 (새로고침해도 유지)
3. 상단 **🛠️ 데모 도구**로 시간여행:
   - `다음 날로 ▶` — 하루 진행 (미완료였던 과거 날은 벌금 확정)
   - `샘플 기록 채우기` — 12일치 기록을 채우고 13일차로 점프
   - `30일 종료 시뮬레이션` — 최종 정산 보기
   - `진행 초기화` / `로그아웃`
4. **모임** 탭 — 나 + 친구들의 30일 진행 그리드, 각자 벌금, 모임통장 합계
5. **정산** 탭 — `한 명 몰아주기` / `잘 지킨 만큼 분배` 규칙별 분배 결과

## 확장 지점 (지금은 미구현)

- **로그인**: `client/src/lib/auth.ts` — 실제 인증(카카오/이메일 등) 붙일 자리
- **실 백엔드/DB**: `client/src/lib/api.ts` + `server/` — 목업을 실 API로 교체
- **앱잠금(Feature B)**: 이번 데모 범위 밖. 순수 웹으로는 불가하며, 추후 Capacitor 네이티브 래핑 필요

## 정산 규칙 (points.ts)

- **한 명 몰아주기(winner)**: 벌금이 가장 적은 사람이 전액 (동점이면 균등)
- **잘 지킨 만큼 분배(proportional)**: 벌금에 반비례해 분배 (잘 지킬수록 많이 받음)

수식은 `client/src/lib/points.ts`의 순수 함수로 분리되어 있어 조정이 쉽습니다.
